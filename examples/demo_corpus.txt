Project Helios is a fictional retrieval benchmark used to demonstrate cartridge-backed question answering.

The benchmark has four key facts:
1. The preferred GPU order is GPU 3, then GPU 5.
2. GPUs 6 and 7 are disallowed.
3. The parent model is Qwen/Qwen3-4B.
4. Smoke runs use offline Weights & Biases, while full runs require online Weights & Biases.

Project Helios also defines three operational rules:
- The corpus should be compressed once into a cartridge and then reused for repeated questions.
- The same single GPU should not host vLLM and local Hugging Face inference at the same time.
- A valid demo should compare cartridge-backed answers against full-context answers.

For clarity, the preferred GPU order is repeated here: GPU 3 first, GPU 5 second. GPUs 6 and 7 remain disallowed.
